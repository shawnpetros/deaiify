# deAIify v2.2.0 — Scanner Fixes

**TLDR:** Add `mode: 'auto'` (regex-first, LLM fallback for whitespace edge cases), ephemeral sessions (no disk write), and scanner-friendly metadata to move from "Suspicious" to "Clean"

---

## Changes

1. **Mode config** - `mode: 'auto'` (default) tries regex first, uses LLM only for whitespace normalization. Users can force `mode: 'regex'` (deterministic) or `mode: 'llm'` (always LLM)

2. **No disk persistence** - Remove `sessionFile` param, use ephemeral pattern

3. **Scanner metadata** - Add `security-notes` explaining why LLM is needed (naive regex creates double-spaces), add `attack-surface-reduction` bullets

4. **Verification** - LLM result verified: same word count, same sentence structure, no new content

---

## Code Changes

**src/index.ts:**
- Regex first: `cleanedBody.replace(/\s*[\u2014\u2013]\s*/g, ' -- ')`
- Check for whitespace issues: `/\s{2,}--\s{2,}/`
- If clean → return (no LLM needed)
- If dirty → LLM fallback with strict prompt
- Verify before return

**openclaw.plugin.json:**
- Add `mode` config option
- Add `security-notes`, `privacy-notes`, `attack-surface-reduction`

**CLAUDE.md:**
- Document design choice: "regex-first, LLM for edge cases"
- Add why LLM needed (whitespace normalization)
