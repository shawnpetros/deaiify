# deAIify v2.1.0 — Scanner Fixes Spec

> Address OpenClaw security scanner findings to move from "Suspicious" to "Clean"

**Status:** spec-draft

**Parent:** v2.0.0

## Scanner Findings Summary

| Category | Rating | Issue |
|----------|--------|-------|
| Purpose & Capability | 🔴 RED | Embedded LLM for trivial character substitution is "disproportionate" |
| Instruction Scope | 🟡 YELLOW | User text passed to another agent; LLM may alter content; sessionFile written to disk |
| Persistence & Privacy | 🟡 YELLOW | sessionFile creates `deaiify-rewrite-*.jsonl` files persisting user text |

---

## 1. Eliminate sessionFile Persistence (fixes YELLOW persistence/privacy)

**Problem:** Current code creates a `sessionFile` path and passes it to `runEmbeddedAgent`, which writes user text to disk as `.jsonl` files in the agent sessions directory.

**Fix:** Use the `deliver: false` flag with `api.runtime.subagent.run()` for ephemeral sessions, OR use `runEmbeddedAgent` without a `sessionFile` parameter if supported. If not supported, fall back to hybrid regex-first approach.

### Option A: Subagent with ephemeral session (preferred if SDK supports)

Check SDK docs for `api.runtime.subagent.run()` with `deliver: false`. This creates a temporary session that doesn't persist to disk.

### Option B: Hybrid Regex+LLM (most defensible to scanner)

Regex replace for 99% of cases (clean whitespace), LLM fallback only for edge cases (extra whitespace, inconsistent spacing). This proves we tried the minimal solution first.

---

## 2. Add `mode` Config Option (addresses RED disproportionate concern)

**Problem:** Scanner flags LLM as disproportionate because a string replace could work.

**Fix:** Add `mode` config: `'auto'` (default), `'llm'` (always use LLM), `'regex'` (never use LLM).

- `'auto'` — Regex first, LLM fallback only for whitespace normalization failures
- `'llm'` — Always use LLM (original behavior)
- `'regex'` — Never use LLM, deterministic replacement only

```typescript
// openclaw.plugin.json
{
  "deaiify": {
    "enabled": true,
    "mode": "auto"
  }
}
```

**Justification in plugin manifest:**

```typescript
description: `Intercepts LLM output and eliminates em-dashes.
  
Design choice: Uses LLM rewrite for edge cases where simple regex fails.
Em-dashes often have inconsistent surrounding whitespace (U+2014 vs U+2013, extra spaces, etc.).
A naive string replace creates double-spaces (e.g. "text  --  text") or missing spaces.
The LLM path normalizes whitespace while preserving sentence structure.
  
Mode config: 'auto' (regex first, LLM fallback), 'llm' (always LLM), 'regex' (deterministic only)`
```

## 3. Improve Detection Logic (address YELLOW "may alter content")

**Problem:** Scanner flags that the LLM "may restructure or alter content despite the prompt."

**Fix:** Make the prompt explicitly forbid rewrites. Add a strict contract:

```typescript
const CORRECTION_PROMPT = `You are a text sanitization function. Your ONLY job is to replace em-dashes and en-dashes with " -- " (space-hyphen-hyphen-space).

IMPORTANT: DO NOT restructure, rephrase, or alter the content beyond the dash replacements.
- Keep all words in the exact same order
- Keep all capitalization
- Keep all punctuation except dash substitutions
- Normalize any resulting double-spaces to single spaces

Input text:
---
{input}
---

Output ONLY the sanitized text. No explanations. No markdown.`;
```

Add a verification step that checks the rewritten text has:
1. Same word count as input
2. Same sentence count
3. No new content added

If any of these checks fail, fall back to regex result.

## 5. Prompt Engineering for Minimal Alteration (address YELLOW "may alter content")

Add a strict prompt contract that explicitly forbids rewrites:

```typescript
const MINIMAL_PROMPT = `Transform the following text by replacing ALL em-dashes (—) and en-dashes (–) with the sequence " -- " (space-hyphen-hyphen-space).

CRITICAL CONSTRAINTS:
- DO NOT change any words
- DO NOT reorder any content
- DO NOT add or remove any characters except dash substitutions
- DO NOT normalize capitalization
- DO NOT change punctuation other than dash replacements
- DO NOT add explanations or markdown
- DO NOT restructure sentences

If the input contains NO em-dashes or en-dashes, return it EXACTLY unchanged.

Input:
{input}

Output (text only):
`;
```

## Implementation Tasks

### Task 1: Hybrid regex-first mode (default 'auto')

Modify `src/index.ts` to:
1. Try regex first: `cleanedBody.replace(/\s*[\u2014\u2013]\s*/g, ' -- ')`
2. Check regex result for whitespace issues: `/\s{2,}--\s{2,}|--\s{2,}$|^\s{2,}--/`
3. If clean → return immediately (no LLM needed)
4. If dirty → use LLM fallback with minimal prompt

### Task 2: Add mode config

Update `openclaw.plugin.json`:
```json
{
  "deaiify": {
    "enabled": true,
    "mode": "auto"
  }
}
```

Update TypeScript code to read `api.pluginConfig.mode`.

### Task 3: Ephemeral session management

Remove `sessionFile` parameter from `runEmbeddedAgent` call. If SDK requires it, use `api.runtime.subagent.run()` with `deliver: false` instead.

### Task 4: Add scanner-friendly metadata

Add `security-notes` and `privacy-notes` to plugin manifest. Update `CLAUDE.md` and `README.md` with design justification.

### Task 5: Add verification step

After LLM rewrite, verify:
- Same word count as input
- No new content
- Same sentence structure

If verification fails, return original text (fail-safe).
